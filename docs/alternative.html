Features

Built for agent-led workflows

Agent PM keeps your AI agents (e.g. Cursor CLI) aligned with your projects. Create work items, assign them to agents, and optionally require approval before work begins.

📋

Work items

Create and assign tasks with title, description, project, and agent. Optional approval gate so you control when work starts.

📊

Kanban boards

Visualize projects with columns. Drag and drop cards. Same data powers the API, web UI, and CLI.

📬

Inbox

Agents see assigned work in an inbox. Require approval so items stay in inbox until you approve them.

⌘

CLI

Use pm to create work items and assign agents from the terminal. Config via .pmrc.json.

🖥

Desktop app

Tauri-based native app. Same React UI; runs against the API. Optional daemon for always-on.

🔌

API

REST API on port 38472. One backend for web, desktop, and CLI. Install as daemon (systemd, launchd, or Windows Task).

Run it your way

Web app, desktop app, or CLI

One product, three ways to use it. The API runs on port 38472; the same data powers the GUI and the CLI.

🌐

Web app

Run the full UI in your browser. Start the API and Vite, then open the app.

npm run dev → localhost:5173
🖥

Desktop app

Tauri desktop app. Same React UI; optional daemon for always-on.

npm run tauri:dev
⌘

CLI

Create work items and assign agents from the terminal.

pm wi --create --title "Task" …

Get started

Quick start

The API server runs on port 38472 by default. Install as a daemon for background use.

$ pm wi --create --title "Fix login bug" --summary "Reproduce and fix" --project "My Project" --agent "Backend Agent"
$ pm wi --create --title "Deploy to prod" --project "My Project" --agent "DevOps Agent" --require-approval
Browser: npm run dev → localhost:5173 Desktop: npm run tauri:dev
