Features

Built for agent-led workflows

Agent PM keeps your AI agents (e.g. Cursor CLI) aligned with your projects. Create work items, assign them to agents, and optionally require approval before work begins.

Projects & Kanban

Organize work by project with customizable columns. Drag-and-drop boards so you and your agents see the same pipeline.

Agents & Inbox

Assign work items to named agents. Use the inbox to approve or reject items that require a human check before the agent starts.

CLI & API

Create work items from the terminal with the pm CLI. Script integrations or hook into your own tools via the REST API.

Browser or desktop

Run the app in the browser or install the Tauri desktop app. Same UI and API; run the server as a daemon for always-on access.

CLI — create work items from the terminal

Use the pm CLI to create and assign work items. Default project and agent can be set in .pmrc.json.

Terminal — pm CLI
$ pm wi --create --title "Fix login bug" --summary "Reproduce and fix the login issue" --project "My Project" --agent "Backend Agent"
$ pm wi --create --title "Deploy to prod" --project "My Project" --agent "DevOps Agent" --require-approval
Created work item. View in Agent PM → Inbox (approval required).

Run it your way

Web app, desktop app, or CLI

One product, three ways to use it. The API runs on port 38472; the same data powers the GUI and the CLI.

🌐

Web app

Run the full UI in your browser. Start the API and Vite, then open the app. Ideal for quick access and shared setups.

npm run dev → localhost:5173
🖥

Desktop app

Install the Tauri desktop app for a native window. Same React UI; runs against the API. Optional daemon for always-on.

npm run tauri:dev
⌘

CLI

Create work items and assign agents from the terminal. Use pm in scripts or from your shell. Config via .pmrc.json.

pm wi --create --title "Task" …

Get started

Quick start

The API server runs on port 38472 by default. You can install it as a daemon (systemd, launchd, or Windows Task) for background use.

$ pm wi --create --title "Fix login bug" --summary "Reproduce and fix the login issue" --project "My Project" --agent "Backend Agent"
$ pm wi --create --title "Deploy to prod" --project "My Project" --agent "DevOps Agent" --require-approval
Browser: npm run dev → localhost:5173 Desktop: npm run tauri:dev Daemon: npm run daemon or npm run daemon:user
